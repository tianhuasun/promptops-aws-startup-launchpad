# Judge Guide

## What This Project Is

PromptOps AWS Startup Launchpad is a production-ready AWS Prompt Kit for the DoraHacks AWS Prompt the Planet Challenge.

It gives an AI coding agent a complete, verbatim master prompt for generating a reviewable AWS MVP infrastructure launch package. The primary scenario is a Next.js frontend, FastAPI backend, PostgreSQL database, S3 static assets, CloudFront CDN, Secrets Manager, CloudWatch logs, AWS Budgets, Terraform, and AWS App Runner.

The goal is to help founders, indie hackers, student builders, and developers turn product requirements into a secure, low-cost, auditable AWS infrastructure plan before any real deployment happens.

## What This Project Is Not

- Not a live AWS deployment.
- Not a chatbot wrapper.
- Not a generic README-only prompt.
- Not a tool that asks for real AWS keys.
- Not a project that creates AWS resources by default.

## Why It Fits AWS Prompt The Planet

The public challenge emphasis is on complete prompts, context and documentation, AWS services, and AWS best practices. This repository is structured around those criteria:

- The complete prompt is in `prompt/master-prompt.md`.
- The supporting prompt contract is in `prompt/`.
- Context and review documentation are in `docs/`.
- Expected outputs are in `examples/`.
- AWS service coverage and best-practice mapping are in `terraform/` and `docs/AWS_WELL_ARCHITECTED_MAPPING.md`.
- Local validation scripts are in `scripts/`.

## 3-Minute Review Path

1. Read the top of `README.md`.
2. Open `prompt/master-prompt.md`.
3. Check `prompt/user-input-template.md`.
4. Skim `examples/nextjs-fastapi-postgres/expected-output/architecture.md`.
5. Open `docs/HACKATHON_SUBMISSION.md`.
6. Run `bash scripts/test-all.sh` if local validation is desired.

## 10-Minute Deeper Review Path

1. Read `prompt/master-prompt.md` from start to finish.
2. Compare the required output contract with `prompt/output-schema.md`.
3. Inspect the safety rules in `prompt/safety-rules.md`.
4. Review the main expected output under `examples/nextjs-fastapi-postgres/expected-output/`.
5. Inspect the Terraform composition in `terraform/examples/nextjs-fastapi-postgres/`.
6. Review the AWS Well-Architected mapping in `docs/AWS_WELL_ARCHITECTED_MAPPING.md`.
7. Review security and cost controls in `docs/SECURITY_BASELINE.md` and `docs/COST_CONTROL.md`.
8. Run `bash scripts/validate-docs.sh` and `bash scripts/test-all.sh`.

## How To Validate Locally

Run:

```bash
bash scripts/validate-docs.sh
bash scripts/test-all.sh
```

If Terraform is installed, the Terraform script checks formatting. If Terraform is not installed, Terraform validation is skipped with a clear message.

## Safety And Non-Deployment Note

This repository does not create AWS resources during validation. It does not require real AWS credentials. It does not include real AWS Account IDs, ARNs, domains, Hosted Zone IDs, Certificate ARNs, tokens, passwords, or `.env` files.

The prompt requires human approval before `terraform apply`, `terraform destroy`, DNS changes, production database changes, secret rotation, budget increases, or high-cost service changes.

## Current Limitations

- The project is a prompt kit and reviewable infrastructure example, not a running AWS environment.
- Terraform examples are intended for static review and formatting validation before any real deployment.
- Cost estimates generated by the prompt should be treated as planning ranges, not billing guarantees.
- The repository is designed for direct GitHub review. A video is not part of the primary review path.
- The DoraHacks project page is created through the DoraHacks submission workflow and is not represented as a repository placeholder.
